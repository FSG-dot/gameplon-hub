# 핑거스타일 TAB 정확도 개선 테스트 시나리오

이 문서는 `영상 → 자동 TAB 생성` 결과의 **정확도(음정/타이밍/플레이어 동기화)**를 반복 측정하기 위한 실무형 체크리스트입니다.

---

## 1) 테스트 목표 (Pass/Fail 기준)

### A. 기능 동작
- [ ] 영상 업로드 후 분석 버튼이 활성화된다.
- [ ] 분석 완료 후 TAB 노트가 렌더링된다.
- [ ] 영상 재생/일시정지/시크(seek) 시 TAB 스크롤이 연동된다.

**Pass 조건**: 3개 항목 모두 정상.

### B. 음정 정확도 (Pitch Accuracy)
- [ ] 기준 구간(단음 위주)에서 생성된 프렛이 실제 악보 대비 일치율 80% 이상.
- [ ] 오픈 스트링(0프렛) 인식 누락률 20% 이하.
- [ ] 고음역(12프렛 이상) 오검출률 25% 이하.

**Pass 조건**: 3개 항목 중 2개 이상 만족 + 전체 음정 일치율 75% 이상.

### C. 타이밍 정확도 (Onset Timing)
- [ ] 실제 발음 시점 대비 자동 이벤트 평균 오차(MAE) 120ms 이하.
- [ ] 이벤트 과다검출(중복 onset) 비율 15% 이하.
- [ ] 이벤트 누락 비율 20% 이하.

**Pass 조건**: 3개 항목 중 2개 이상 만족.

### D. 사용성
- [ ] 60초 영상 분석 시간 20초 이하(일반 노트북 기준).
- [ ] 브라우저 멈춤/크래시 없이 분석 완료.

**Pass 조건**: 2개 모두 만족.

---

## 2) 샘플 영상 세트 구성 (최소 6개)

서로 다른 난이도와 주법을 섞어야 개선 방향이 명확해집니다.

1. **S1-단음 아르페지오(저난도)**: 코드톤 분산, 템포 70~90 BPM
2. **S2-단음+베이스 교차(중난도)**: 저음/고음 교대
3. **S3-슬라이드 포함(중난도)**: 포지션 이동 많음
4. **S4-해머온/풀오프(중난도)**: 레가토 다수
5. **S5-다성음(고난도)**: 동시에 2~3음 발음
6. **S6-퍼커시브 포함(고난도)**: 바디 히트/뮤트 포함

각 샘플은 20~40초 길이 권장.

---

## 3) 테스트 실행 절차

1. 앱 실행
   - `python3 -m http.server 4173 --bind 0.0.0.0 --directory /workspace/gameplon-hub`
2. 브라우저 접속
   - `http://localhost:4173`
3. 샘플 영상 1개 업로드
4. BPM 입력 (원본 곡 평균 BPM)
5. `영상 분석 후 악보 생성` 클릭
6. 생성된 TAB를 기준 악보/수동 라벨과 비교
7. 아래 결과표에 수치 기록
8. S1~S6 반복

## 4) 결과 기록 템플릿 (복붙용)

아래 표를 샘플마다 채워주세요.

| 샘플ID | 길이(초) | 총 기준 음수 | 정탐(TP) | 오탐(FP) | 누락(FN) | 음정 일치율 | Onset MAE(ms) | 분석 시간(초) | Pass/Fail | 비고 |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---|---|
| S1 |  |  |  |  |  |  |  |  |  |  |
| S2 |  |  |  |  |  |  |  |  |  |  |
| S3 |  |  |  |  |  |  |  |  |  |  |
| S4 |  |  |  |  |  |  |  |  |  |  |
| S5 |  |  |  |  |  |  |  |  |  |  |
| S6 |  |  |  |  |  |  |  |  |  |  |

### 계산식
- 음정 일치율(%) = `TP / (TP + FP + FN) × 100`
- 과다검출 비율(%) = `FP / (TP + FP) × 100`
- 누락 비율(%) = `FN / (TP + FN) × 100`

## 5) 개선 우선순위 가이드 (Fail 발생 시)

### 증상 1: 노트가 너무 많이 찍힘 (FP 과다)
- 에너지 변화 임계값(`flux threshold`) 상향
- 최소 노트 간격(`minDistanceSeconds`) 상향
- 저레벨 잡음 구간 게이트 추가

### 증상 2: 노트가 많이 빠짐 (FN 과다)
- 임계값 하향
- 프레임/홉 크기 재조정
- 입력 오디오 정규화(peak normalize) 도입

### 증상 3: 프렛/줄 매핑이 어색함
- 단순 최소 프렛 선택 대신
  - 직전 포지션 기반 비용함수(왼손 이동 최소화)
  - 개방현 선호/비선호 옵션

### 증상 4: 다성음에서 정확도 급락
- 단일 피치 추정 → 다중 피치 추정(스펙트럼 피크 상위 N개)
- 동시에 2개 음까지 제한적 허용

## 6) 릴리즈 게이트 (최종 합격 기준)

다음 조건을 모두 만족하면 "정확도 개선 완료"로 판단:

- [ ] S1~S6 평균 음정 일치율 78% 이상
- [ ] S1~S4 평균 Onset MAE 110ms 이하
- [ ] S5~S6에서도 앱 크래시/멈춤 없음
- [ ] 회귀 테스트(기존 통과 샘플) 재실행 시 성능 하락 없음

---

## 7) 빠른 수동 체크리스트 (5분 스모크 테스트)

- [ ] 업로드 직후 버튼 활성화 확인
- [ ] 분석 완료 문구 노출 확인
- [ ] 재생 시 TAB가 플레이헤드 기준으로 움직임
- [ ] 시크 후 TAB 위치 즉시 동기화
- [ ] 페이지 새로고침 후 재업로드/재분석 가능

위 5개 중 1개라도 실패하면 배포 보류.
